# Naia Specifications

This directory contains the normative specifications for Naia.

## Directory Structure

- **`contracts/`** - Sacred behavior contracts (source of truth)
  - These define what Naia MUST do
  - Human-authored, carefully maintained
  - Changes require careful review

- **`generated/`** - Derived artifacts (regenerated by tooling)
  - `NAIA_SPECS.md` - All contracts bundled into one file
  - `CONTRACT_REGISTRY.md` - Index of all contract IDs
  - `TRACEABILITY.md` - Contract-to-test mapping
  - `GAP_ANALYSIS.md` - Coverage gaps

- **`spec_tool.sh`** - Tooling for spec management

## Common Commands

```bash
./specs/spec_tool.sh coverage      # Check test coverage
./specs/spec_tool.sh lint          # Validate spec format
./specs/spec_tool.sh bundle        # Regenerate NAIA_SPECS.md
./specs/spec_tool.sh registry      # Regenerate CONTRACT_REGISTRY.md
./specs/spec_tool.sh traceability  # Regenerate test mapping
```
