# Specs Directory Refactoring Plan

**Goal:** Separate sacred behavior contracts from generated artifacts and tooling.

**Rationale:** The current `specs/` directory mixes source-of-truth contract documents with generated artifacts, causing confusion about what is sacred vs. derived.

### Phase 4: Create New specs/README.md

Create `specs/README.md` explaining the new structure:

```markdown
# Naia Specifications

This directory contains the normative specifications for Naia.

## Directory Structure

- **`contracts/`** - Sacred behavior contracts (source of truth)
  - These define what Naia MUST do
  - Human-authored, carefully maintained
  - Changes require careful review

- **`generated/`** - Derived artifacts (regenerated by tooling)
  - `NAIA_SPECS.md` - All contracts bundled into one file
  - `CONTRACT_REGISTRY.md` - Index of all contract IDs
  - `TRACEABILITY.md` - Contract↔test mapping
  - `GAP_ANALYSIS.md` - Coverage gaps

- **`spec_tool.sh`** - Tooling for spec management

## Common Commands

```bash
./specs/spec_tool.sh coverage      # Check test coverage
./specs/spec_tool.sh lint          # Validate spec format
./specs/spec_tool.sh bundle        # Regenerate NAIA_SPECS.md
./specs/spec_tool.sh traceability  # Regenerate test mapping
```
```

### Phase 5: Update spec_tool.sh

The script uses `$SCRIPT_DIR` to find files. Update it to use the new paths:

**Key changes needed in `specs/spec_tool.sh`:**

1. **Line ~21:** Add contracts and generated directory variables:
   ```bash
   SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   CONTRACTS_DIR="$SCRIPT_DIR/contracts"
   GENERATED_DIR="$SCRIPT_DIR/generated"
   ```

2. **`get_spec_files()` function (~line 55-70):** Change to read from `$CONTRACTS_DIR`:
   ```bash
   # Change: find "$SCRIPT_DIR" -maxdepth 1 -name '*.md'
   # To:     find "$CONTRACTS_DIR" -maxdepth 1 -name '*.md'
   ```

3. **`cmd_bundle()` function (~line 144):** Update output path:
   ```bash
   # Change: local output_file="${1:-$SCRIPT_DIR/NAIA_SPECS.md}"
   # To:     local output_file="${1:-$GENERATED_DIR/NAIA_SPECS.md}"
   ```

4. **`cmd_registry()` function (~line 455):** Update output path:
   ```bash
   # Change: local output_file="${1:-$SCRIPT_DIR/CONTRACT_REGISTRY.md}"
   # To:     local output_file="${1:-$GENERATED_DIR/CONTRACT_REGISTRY.md}"
   ```

5. **`cmd_traceability()` function:** Update output path similarly.

6. **`cmd_coverage()` function (~line 657):** Update registry path:
   ```bash
   # Change: grep ... "$SCRIPT_DIR/CONTRACT_REGISTRY.md"
   # To:     grep ... "$GENERATED_DIR/CONTRACT_REGISTRY.md"
   ```

7. **Template checks:** Update references to `1_template.md` → `_template.md`

8. **Cross-reference validation:** Update to check `$CONTRACTS_DIR`

### Phase 6: Update Cross-References in Contract Files

Several contract files reference other specs. Update these paths:

**Files with cross-references:**
- `specs/contracts/13_server_events_api.md` (lines 8-12)
- `specs/contracts/14_client_events_api.md` (lines 25-29)
- `specs/contracts/15_world_integration.md` (lines 18-28)

**Pattern to change:**
```
# Old: `specs/entity_replication.md`
# New: `entity_replication.md` (relative within contracts/)
#  OR: `specs/contracts/8_entity_replication.md` (full path)
```

**Recommended approach:** Use relative references without path prefix since they're all in the same directory now. Example:
```markdown
# Old
- `specs/entity_replication.md`
- `specs/entity_scopes.md`

# New
- `8_entity_replication.md`
- `7_entity_scopes.md`
```

## Rollback Plan

If something goes wrong, git can restore the original state:

```bash
git checkout -- specs/
git clean -fd specs/contracts specs/generated
```

## Notes

- The `spec_tool.sh` command interface stays the same (`./specs/spec_tool.sh coverage` etc.)
- Test files do NOT need updating - they only contain contract IDs like `[entity-authority-11]`, not file paths
- The numbered prefix on contract files (2_, 3_, etc.) is preserved for ordering
- Generated files in `specs/generated/` could optionally be added to `.gitignore` if you prefer to regenerate on CI
